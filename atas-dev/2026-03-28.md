# Ata de Desenvolvimento - 2026-03-28

## Resumo do dia

Hoje o projeto `corrija_me_pt_br` consolidou a base do corretor local em pt-BR com backend Node.js, extensão Chrome, dicionário próprio e primeiras regras de contexto em JSON.

## O que deu certo

- A instalação local para Linux e Windows foi simplificada com pacotes portáteis.
- O backend passou a funcionar com porta dinâmica, evitando conflito com serviços já rodando.
- A extensão conseguiu evoluir de um popup técnico para um fluxo mais próximo de corretor real.
- Foi criada uma página de teste própria em `test/index.html`.
- O dicionário local foi estruturado em `data/dictionary/`.
- As regras de contexto passaram a morar em `data/rules/context_rules.json`.
- O projeto ganhou validação com TypeScript e ESLint.
- O fluxo de release foi consolidado em `npm run release`.
- A página de teste foi preparada para publicação via GitHub Pages.

## O que deu errado

- Várias vezes o pacote instalado ficou defasado em relação ao código do repositório.
- Houve confusão entre alterar código e regenerar os arquivos finais de instalação.
- A leitura de dados no pacote portátil exigiu ajustes para não depender de descoberta implícita de arquivos.
- O comportamento visual da extensão em `input` e `textarea` ainda ficou atrás do esperado em alguns momentos.
- Algumas regras de contexto iniciais estavam fracas, ambíguas ou simplesmente ruins.
- O fluxo de edição do `context_rules.json` gerou retrabalho por falta de um processo claro de manutenção.

## Conquistas importantes

- Já existe um corretor ortográfico local funcional.
- Já existe um dicionário próprio com mais de 40 mil entradas úteis.
- Já existe suporte inicial a regras por contexto.
- Já existe uma arquitetura que permite crescer sem depender do LanguageTool.
- Já existe uma base técnica boa o suficiente para sair do “protótipo improvisado” e entrar em “produto em construção”.

## Status resumido

- Dicionário útil normalizado: cerca de 40.760 palavras.
- Regras atuais em `context_rules.json`: 88.
- Fluxo de release: funcionando.
- Página de teste: pronta.
- GitHub Pages: preparado.
- Próximo gargalo real: qualidade linguística e manutenção dos dados.

## Visão de continuidade

O projeto já resolveu boa parte dos problemas de infraestrutura, empacotamento e base técnica. O próximo salto não depende mais tanto de engenharia do instalador, e sim de qualidade do motor linguístico:

- aumentar a qualidade do ranking de sugestões
- revisar e expandir as regras de contexto
- aumentar o dicionário com mais cobertura real de pt-BR
- reduzir falsos positivos e falsos negativos
- reforçar a experiência visual da extensão

## O que deveríamos fazer amanhã

1. Revisar o `context_rules.json` e remover regras fracas, ambíguas ou mal formuladas.
2. Criar testes automáticos específicos para regras de contexto.
3. Separar as regras por categoria no futuro, para evitar um único arquivo gigante.
4. Criar o processo de entrada de novas palavras no dicionário com revisão mínima.
5. Melhorar o ranking das sugestões ortográficas para mostrar primeiro a correção mais provável.
6. Gerar uma nova release completa depois de revisar regras e dicionário.
